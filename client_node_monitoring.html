ℹ️ Monitoring Logic Explained

💓 Heartbeat Mechanism

Browser sends periodic heartbeats every 30-60 seconds. This proves the browser/client is still connected, even if user is idle.

Endpoint: /api/heartbeat/

⚡ User Activity Tracking

Tracks real interactions: page requests, exam submissions, form data. Different from heartbeat - only sent when user actively does something.

Tracked from: RequestLog, ExamAttempt, navigation

🟢 ACTIVE Status

User is actively using the system. Last activity detected within {{ timeout_seconds }} seconds. Node is healthy and responsive.

🟡 INACTIVE Status

Browser/client is connected (heartbeat received) but user is idle. No recent activity detected. Connection is alive.

🔴 NODE FAILURE Status

Client is unreachable. No heartbeat received for extended period. Likely causes: PC shutdown, WiFi disconnected, browser closed, network error.

✅ Real Data Only

No fake values or simulation. All data comes from actual database records: RequestLog, ExamAttempt, ActiveConnection, Student profiles.
